API Documentation

Google Drive AI Organizer REST API 레퍼런스

Base URL: http://127.0.0.1:8888

Authentication

GET /auth/google

Google OAuth 2.0 로그인 시작

Response

302 Redirect to Google OAuth consent page
GET /auth/callback

OAuth 콜백 처리 및 JWT 토큰 발급

Query Parameters

Parameter Type Required Description
code string Required Google OAuth authorization code

Response

{
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "user": {
        "email": "user@example.com",
        "name": "User Name"
    }
}
GET /auth/me

현재 인증된 사용자 정보 조회

Headers

Authorization: Bearer {JWT_TOKEN}

Response

{
    "email": "user@example.com",
    "name": "User Name",
    "authenticated": true
}

Files

GET /files/list

Google Drive 파일 목록 조회

Query Parameters

Parameter Type Required Description
include_folders boolean Optional 폴더 포함 여부 (default: false)
page_size integer Optional 페이지 크기 (default: 100, max: 1000)
page_token string Optional 다음 페이지 토큰

Response

{
    "files": [
        {
            "id": "1234567890",
            "name": "document.pdf",
            "mimeType": "application/pdf",
            "modifiedTime": "2025-08-08T10:00:00Z",
            "size": "1048576",
            "parents": ["folder_id"]
        }
    ],
    "nextPageToken": "token123"
}

POST /files/move

파일을 다른 폴더로 이동

Request Body

{
    "file_id": "1234567890",
    "folder_id": "0987654321"
}

Response

{
    "success": true,
    "file": {
        "id": "1234567890",
        "name": "document.pdf",
        "parents": ["0987654321"]
    }
}
DELETE /files/{file_id}

파일 삭제

Path Parameters

Parameter Type Required Description
file_id string Required 삭제할 파일 ID

Response

{
    "success": true,
    "message": "File deleted successfully"
}

Folders

POST /folders/create

새 폴더 생성

Request Body

{
    "name": "New Folder",
    "parent_id": "root"  // optional, default: root
}

Response

{
    "folder": {
        "id": "folder123",
        "name": "New Folder",
        "mimeType": "application/vnd.google-apps.folder"
    }
}

Organization

POST /organize/start

자동 정리 작업 시작

Request Body

{
    "auto_mode": true,
    "confidence_threshold": 0.8,  // 0.5 - 1.0
    "max_files": 100,
    "create_folders": true
}

Response

{
    "job_id": "job_20250808_123456",
    "status": "started",
    "estimated_time": 180  // seconds
}
GET /organize/status/{job_id}

정리 작업 진행 상황 조회

Path Parameters

Parameter Type Required Description
job_id string Required 작업 ID

Response

{
    "job_id": "job_20250808_123456",
    "status": "in_progress",  // started, in_progress, completed, failed
    "progress": 65.5,
    "processed_files": 65,
    "total_files": 100,
    "organized_files": 58,
    "errors": []
}

Error Responses

모든 API는 다음과 같은 형식의 에러 응답을 반환할 수 있습니다:

{
    "error": {
        "code": "UNAUTHORIZED",
        "message": "Authentication required",
        "details": "JWT token is missing or invalid"
    }
}

Error Codes

HTTP Status Error Code Description
401 UNAUTHORIZED 인증 필요
403 FORBIDDEN 권한 없음
404 NOT_FOUND 리소스를 찾을 수 없음
429 RATE_LIMIT_EXCEEDED 요청 제한 초과
500 INTERNAL_ERROR 서버 내부 오류

Rate Limiting

API는 다음과 같은 요청 제한이 적용됩니다:

  • 인증된 요청: 분당 60회
  • 파일 작업: 분당 30회
  • 정리 작업: 시간당 10회

응답 헤더에서 제한 정보를 확인할 수 있습니다:

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1628444400
